### Phase 5: 리스크 관리 자동화 (3일)

**목표**: 자동 리스크 모니터링 및 대응

**기능**:
1. **MDD 모니터링**
   ```python
   if current_mdd < -3.0:
       alert("⚠️ MDD -3% 초과!")
       if current_mdd < -5.0:
           alert("🚨 긴급! MDD -5% 초과, 포지션 자동 청산 권장")
           # 자동 청산 옵션
   ```

2. **연속 손실 감지**
   ```python
   if consecutive_losses >= 3:
       alert("⚠️ 연속 3회 손실 발생")
       reduce_position_size(0.3)  # 포지션 50% → 30%
   ```

3. **변동성 경고**
   ```python
   if daily_volatility > 5.0:
       alert("📉 높은 변동성 감지: {:.2f}%".format(daily_volatility))
       # 신규 진입 제한
   ```

4. **포트폴리오 건강도 점수**
   ```python
   health_score = (
       0.3 * sharpe_score +
       0.3 * mdd_score +
       0.2 * win_rate_score +
       0.2 * profit_factor_score
   )
   # 100점 만점
   # 80+: 우수, 60-80: 양호, 40-60: 주의, 40-: 위험
   ```

**자동 대응**:
- MDD -5% 초과: 포지션 자동 청산 (옵션)
- 연속 손실 3회: 포지션 크기 축소 (70% → 30%)
- 건강도 40점 미만: 1주일 거래 중단

**Output**:
- `risk_manager.py`: 리스크 관리 모듈
- `auto_response.py`: 자동 대응 로직
- `config_risk.json`: 리스크 설정

### Phase 6: 원클릭 거래 (3일)

**목표**: 텔레그램 알림에서 바로 거래 실행

**기능**:
1. **텔레그램 인라인 버튼**
   ```
   🟢 [매수 시그널]
   가격: 95,500,000원
   포지션: 70% (7,000,000원)

   [매수 실행] [무시] [자세히]
   ```

2. **거래 확인 플로우**
   ```
   사용자: [매수 실행] 클릭
   봇: "95,500,000원에 7,000,000원 매수하시겠습니까?"
       [확인] [취소]
   사용자: [확인] 클릭
   봇: "주문 전송 중..."
       "✅ 매수 완료! 체결가: 95,520,000원"
   ```

3. **주문 상태 추적**
   - 주문 대기 중
   - 체결 중 (부분 체결)
   - 체결 완료
   - 체결 실패 (에러 메시지)

4. **안전 장치**
   - 2단계 확인 (시그널 → 확인 → 실행)
   - 가격 범위 체크 (시그널 대비 ±1% 초과 시 경고)
   - 일일 거래 한도 (최대 3회)
   - 긴급 중단 버튼

**Upbit API 연동**:
```python

    market='KRW-BTC',
    price=95500000,
    volume=0.0733  # 7,000,000 / 95,500,000
)

# 매도 주문
order = upbit.sell_limit_order(
    market='KRW-BTC',
    price=100600000,
    volume=0.0733
)
```

**Output**:
- `telegram_trader.py`: 거래 봇
- `upbit_api.py`: Upbit API 래퍼

### Phase 7: 완전 자동 매매 (선택, 2일)

**목표**: 인간 개입 없이 자동 매매 (고급)

**기능**:
1. **자동 매매 모드**
   ```
   텔레그램 명령어:
   /auto_on : 자동 매매 활성화
   /auto_off : 자동 매매 비활성화
   /auto_status : 현재 상태 확인
   ```

2. **자동 실행 로직**
   ```python
   # 매일 09:00 시그널 체크
   signal = strategy.get_signal(df)

   if signal['action'] == 'buy' and auto_mode:
       order = place_buy_order(signal)
       notify_telegram(f"✅ 자동 매수: {order}")

   elif signal['action'] == 'sell' and auto_mode:
       order = place_sell_order(signal)
       notify_telegram(f"✅ 자동 매도: {order}")
   ```

3. **안전 장치 (자동 매매)**
   - 일일 거래 한도: 3회
   - 최대 손실 한도: -5%

   - TP/SL 도달 시 즉시 청산
   - 체결 실패 시 재시도 (최대 3회)

- 법적 책임은 사용자에게

- `auto_trader.py`: 자동 매매 엔진
- `safety_monitor.py`: 안전 장치
- `config_auto.json`: 자동 매매 설정

## 🚀 시작하기

### Quick Start (개발 환경)
```bash
# 1. 저장소 클론
git clone https://github.com/your-repo/bitcoin-trading-bot
cd strategies/v36_visualization_alerts

# 2. 가상환경 생성
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 3. 의존성 설치
pip install -r requirements.txt

# 4. 설정 파일 생성
cp config/config_telegram.json.example config/config_telegram.json
# API 키 입력

# 5. 대시보드 실행
streamlit run visualization/dashboard.py

# 6. 텔레그램 봇 실행 (별도 터미널)
python alerts/telegram_bot.py
```

### Production 배포 (Docker)
```bash
# 1. Docker 이미지 빌드
docker-compose build

# 2. 컨테이너 실행
docker-compose up -d

# 3. 로그 확인
docker-compose logs -f

# 4. 중지
docker-compose down
```
